The Common Simulation Framework Installation Instructions

Author: Jorge Calderon
Release 0.1.TBD
11/19/2014

Requirements

  1. JDK 1.7 recommended (TBD: determine minimum version supported)
  2. Windows, MacOSX, or *nix (The instructions assume Windows for now)

Third Party Software Downloads

Download and install the following software
  1. Repast Simphony (Java version)
  2. (Future, not needed with present version): JADE
  3. Redis
  4. git to pull the code
Add redis-server, redis-cli, and git to your environment path variable.

It is possible run the Common Simulation Framework without the full Repast Simphony IDE, either using the command line or with a general Eclipse distribution. However, the instructions here assume that the Repast Simphony customization of the eclipse IDE will be used to run the Common Simulation framework, the distributed systems (Multi-agent systems) and distributed agent models.

Installation Instructions

  1. Download and install Repast Simphony (Java Version), Redis, and git.
  2. Download the Common Simulation Framework Code (To be moved to GitHub)
    1. Log on to code.seas.harvard.edu with your Harvard id.
    2. Clone this repository: git@code.seas.harvard.edu:jcalderon-thesis/jcalderon-thesis.git
    3. Go to the project page: https://code.seas.harvard.edu/jcalderon-thesis/jcalderon-thesis
    4. Copy the git URL and clone it on your local machine using this command: git clone git@code.seas.harvard.edu:jcalderon-thesis/jcalderon-thesis.git
  3. Install JZombies_CSF_Demo
    1. While this simulation comes with the Repast samples (File-Import-Repast Simphony-Import Repast Example Models), the CSF version is needed as it includes code to connect to the CSF.  The name of the project/directory is JZombies_CSF_Demo.
      1. Temporarily rename the JZombies_CSF_Demo directory.
      2. File-Import-New-Repast Simphony-Select your workspace folder-Add the JZombies_CSF_Demo folder to the Location and make it the project name-Next-Finish.  If you get an error stating that the project "overlaps the location of another project", you probably didn't include the unique directory name for the new project or the folder exists already.  Make sure to temporarily rename the Zombies_CSF_Demo so that a new Repast Simphony project can be created, as the project metadata is not checked into source control.
      3. Copy the contents of your renamed JZombies_CSF_Demo directory into the new directory
      4. Delete the jZombies_CSF_Demo package directory, leaving only the "jzombies" package.
      5. Right click on the project on the left side (package explorer) and click Refresh.
      6. Click the down arrow next to the green "Run" button.  Verify that Repast Simphony/Eclipse has created a run configuration for  JZombies_CSF_Demo simulation.
      7. Click the simulation in the drop down list for JZombies_CSF_Demo.
      8. If you see an error when using Windows, such as "WARNING: Could not open/create prefs root node Software\JavaSoft\Prefs at root 0x80000002. Windows RegCreateKeyEx(...) returned error code 5." in the eclipse console window, you can implement the fix described here, but is not necessary. 
      9. The Repast GUI should pop up after a few seconds.
      10. Run the simulation by clicking on the blue run button.  You should see the image on the right of zombies catching humans.  The ticks should increment by 1s.  You should be able to step one tick at a time.
      11. The Repast "Getting Started guide" is a great place to start for learning Repast.  Even if you don't plan to create your own simulations, browse through the instructions so you learn  the directory structure, parameters, and how to use the GUI.
  4. Open Repast Simphony (Eclipse) and create a separate new Java Project for these projects under the cloned local repo (File-New-Java Project), unselect "Use default location", browse to each directory in the list below. 
    1. CommonSimulationFramework_CommonLibrary
      1. Create a new Java Project for CommonSimulationFramework_CommonLibrary.
    2. CSF_JADE_Controller_Agent
      NOTE      :  The CSF_JADE_Controller_Agent will become a become a JADE agent in the future. For now, it is a standalone Java program that holds mock Human agents for the simulation.
    3. RepastSimulationRunner
  5. Add the third party libraries to CommonSimulationFramework_CommonLibrary. For now, download: https://drive.google.com/file/d/0B8tCWle1gJ9XOXZ4ZS1xSlIzSGM/view?usp=sharing, and extract this lib directory to the CommonSimulationFramework_CommonLibrary/lib directory.  Make sure you don't end up with a /lib/lib directory.  These files aren't in our source control to prevent having to add binaries.  All of this can be simplified in the future using Maven or Grails.
  6. Right click on the CommonSimulationFramework, Right click on Build Path, Click on the libraries tab, select add JARs, and select all of the JARs in the CommonSimulationFramework/lib folder and its subfolders.
  7. Click on the libraries tab, Add External JARs, and select the "repast.simphony.bin_and_src.jar", which is in your {Repast_Installation}/eclipse/plugins/repast.simphony.bin_and_src_2.1.0 directory.
  8. Add all of the External JARS in the following directory: {Repast_Installation}/eclipse/plugins/repast.simphony.runtime_2.1.0/lib
  9. Right click on the CommonSimulationFramework project and add  JZombies_CSF_Demo.  For now there the there is JZombies code that is inside of the common library. In the future this should be moved to a separate project.
  10. The project will show some errors  in the Junit tests (Java unit tests) due to missing Eclipse plugins/software.  It is recommended that the following software be added to Eclipse to get rid of these errors and to make it easier to find your own errors:  Help-Install New Software.  In the "Work with" drop down, select Kepler (this is the verion of Eclipse), Select "Web, XML, Java EE, and OSGI Enterprise Development" and restart.  Alternatively, you can look for just the individual missing components.  In order to get passed the Junit errors, right click on the "test" folder, right click New-Create a new JUnit test, give it a name, Next, select "Add Junit to the build path".  The errors on the Junit tests should go away.
  11. Select the Libraries button and select JUnit if available (see the note about installing the additional software for Eclipse to be able to get JUnit).
  12. Click "OK".
  13. Right click on the JZombies_CSF_Demo project and add the CommonSimulationFramework project.
  14. Right click on JZombies_CSF_Demo, click on project-Build Path.
  15. Select the Libraries button and select JUnit if available (see the note about installing the additional software for Eclipse to be able to get JUnit).
  16. Select add JARs, and select the jars under the folder and subfolders in CommonSimulationFramework//lib.
  17. Select the External JARs button and add all of the JARS from the two red bolded {Repast_Installation}/eclipse lines above.
  18. Follow the same steps for the RepastSimulationRunner and CSF_JADE_Controller_Agent as for the  JZombies_CSF_Demo project.
  19. For the RepastSimulationRunner also make sure to add the JZombies_CSF_Demo project.
  20. For the CSF_JADE_Controller_Agent, don't add the JZombies_CSF_Demo.
  21. Open the launchers folder, right click on RepastSimulationRunner - CSF.launch, right click, open with Editor.  Change the path to the simulation in simulation to your JZombies_Demo path.
  22. Right click on the launcher file again, "Run As", click "RepastSimulationRunner - CSF.launch" .  The simulation runner has started executing.  Next time you should see it in the drop down next to the run button.
  23. You should see an error message initially as Redis has not been started.  The configuration in eclipse is now complete.
  24. Click on that drop down next to the green run, Run Configurations, select the "RepastSimulationRunner - CSF.launch button", select the Arguments tab.  You should see the path to your JZombies_Demo diurectory.
  25. You are ready to start the execution of the simulation run(s) now as described in the next section.

Simulation Execution

  1. Start up all of the Processes
    1. Start the Redis server by opening the launcher folder, selecting the appropriate subfolder for your operating system, and double clicking on "redis-server".   If it opens as a text file, right click it, open with, default editor.   (TODO: Create another version for Linux/MacOS for each of these files). Keep a separate terminal window open for Redis and watch for error messages.  Alternatively, in Windows, Redis can be installed as a service.
    2. Optionally, run a separate terminal window for the redis command line "redis-cli".  This allows you to inspect messages.  For example, if you shut down or pause the distributed system(s), you can checkto see what message the simulation engine (Repast) is trying to send those systems.
      1. Useful commands include "KEYS *" to see all messages, "lpop thechannelname" to pop a value from the list.  Note, that these messaging structures may be changed in the future to apublish/subscribe model, which would require a different command to read the messages.
    3. Navigate to the launchers folder and right click on "RepastSimulationRunner - CSF.launch".   You should see that the RepastSimulationRunner, running the Simulation side of the CSF begins to execute.  After right clicking this .launch file, you should now see this entry as a run configuration in the drop down next to the green run button at the top.  In the future, you can execute this run from there.  If any changes are needed such as a changed program argument (such as a new location of the CSF configuration file), changes directly to the run configuration will not change the .launch file. It would be best to change the .launch file, delete the run configuration, and right click again.
    4. Click on the drop down next to the run button and select RepastSimulationRunner - CSF.  You should see in the console that the Common Simulation Framework is polling the Redis in-memory cache for commands from you, the simulation administrator.
    5. Start up the distributed system(s)
  2. Kick off the Simulation Run(s)
    1. All of the processes are running now.  The distributed system(s) are waiting on the simulation engine and as mentioned already, the simulation engine is waiting on you, the simulation administrator.
    2. Start the simulation run(s) by passing a START_SIMULATION command to the simulation engine.  Double click on"sendStartSimulationCommandFromAdmin.bat"t to do so.
    3. If you terminate the simulation early, make sure to flush the Redis cache by clicking on flushRedisCache.bat.
