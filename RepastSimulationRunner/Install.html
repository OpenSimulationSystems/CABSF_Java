The Common Simulation Framework Installation Instructions

Author: Jorge Calderon
Release 0.1.TBD
11/18/2014

Requirements

  1. JDK 1.7 recommended (TBD: determine minimum version supported)
  2. Repast Simphony (Java version)
  3. (Future, not needed with present version): JADE
  4. Redis
  5. git to pull the code

Installation Instructions

  1. Download and install Repast Simphony (Java Version), Redis, and git.
  2. Download the Common Simulation Framework Code (To be moved to GitHub)
    1. Log on to code.seas.harvard.edu with your Harvard id.
    2. Clone this repository: git@code.seas.harvard.edu:jcalderon-thesis/jcalderon-thesis.git
    3. Go to the project page: https://code.seas.harvard.edu/jcalderon-thesis/jcalderon-thesis
    4. Copy the git URL and clone it on your local machine using this command: git clone git@code.seas.harvard.edu:jcalderon-thesis/jcalderon-thesis.git
  3. Install JZombies_Demo
    1. You can either install all of the sample simulations that come with Repast Simphony, or you can use the JZombies_Demo that is in the CSF repository.  If using the CSF repository version, follow these steps.
      1. File-New-Repast Simphony Project, unselect "Use default location", and select the location of JZombies_Demo from the local repo.
      2. Click the down arrow next to the green "Run" button.  Verify that Repast Simphony/Eclipse has created a run configuration for this simulation.
      3. Click the simulation in the drop down list for JZombies_Demo.
      4. If you see an error such as "WARNING: Could not open/create prefs root node Software\JavaSoft\Prefs at root 0x80000002. Windows RegCreateKeyEx(...) returned error code 5." in the eclipse console window, you can implement the fix described here, but is not necessary. 
      5. Make sure that the simulation runs.  The Repast "Getting Started guide" is a great place to start for learning Repast.  Even if you don't plan to create your own simulations, browse through the instructions so you learn  the directory structure, parameters, and the GUI.
  4. Open Repast Simphony (Eclipse) and create a separate new Java Project for these projects under the cloned local repo (File-New-Java Project), unselect "Use default location", browse to each directory in the list below:
    1. RepastSimulationRunner
    2. CSF_JADE_Controller_Agent
      3. NOTE:  The CSF_JADE_Controller_Agent will become a become a JADE agent in the future. For now, it is a standalone Java program that holds mock Human agents for teh simulation.

Simulation Execution

  1. RepastSimulationRunner programmatically runs the Repast Simphony simulation.  Select the down arrow by the run button and click on "Run Configurations"
  2. Create a new Java Application configuration.  Select  the RepastSimulationRunner project, and "org.simulationsystems.csf.sim.runners.repastS.RepastS_SimulationRunnerMain" for the main class.
  3. Click on arguments, and add the location of your JZombies.rs directory followed by a space and then any text. The text is a placeholder for the CSF configuration file to be read in the future. If this text isn't specified, RepastSimulationRunner will run the simulation programmatically without any of the CSF functionality.  Click "Apply" and then close the window.
  4. Right click on the RepastSimulationRunner project-Build Path-Projects, and select the JZombies project. 
  5. Click on the libraries tab, Add External JARs, and select the "repast.simphony.bin_and_src.jar", which is in your {Repast_Installation}/eclipse/plugins directory.
  6. Add all of the External JARS in the following directory: {Repast_Installation}/eclipse/plugins/repast.simphony.runtime_2.1.0/lib
  7. Click "OK".
  8. Run the RepastSimulationRunner run configuration .  You should see that it is polling the Redis in-memory cache for  commands from you, the simulation administrator.
  9. Navigate to the launchers folder and double click on "startSimulation.bat".  You should see that the RepastSimulationRunner, using the Simulation side of the CSF begins to process the simulation.
